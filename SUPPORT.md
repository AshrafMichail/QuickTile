# Support

## Getting Help

Before opening an issue:

1. Read [README.md](README.md) for build, configuration, and shortcut details.
2. Confirm you are using the latest version of the repository.
3. Reproduce the problem consistently.
4. Check whether an existing issue already covers the same problem.

## Open an Issue For

- Reproducible bugs
- Regressions from a previous version
- Incorrect or outdated documentation
- Clear feature requests with a concrete use case

## Include This Information

When reporting a problem, include:

- Windows version
- QuickTile commit or release version
- Steps to reproduce
- Expected behavior
- Actual behavior
- Relevant `settings.yaml` content if configuration affects the issue
- Screenshots, logs, or short recordings when helpful

## Support Scope

This repository can help with:

- Building QuickTile from source
- Understanding configuration and default shortcuts
- Diagnosing reproducible tiling, focus, layout, or window classification issues

This repository does not guarantee support for:

- General Windows troubleshooting
- Unrelated third-party tools or shell customizations
- One-off private debugging sessions
- Every possible application window type or environment-specific integration

## Security Issues

Do not report suspected security issues in public issues.
If a `SECURITY.md` file is added later, follow that policy instead.
